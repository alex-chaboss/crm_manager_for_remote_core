# Контракт кастомного `project-widget`

Динамическая загрузка произвольного Python из каталога **`Projects/<id>/project-widget/`** выполняется на машине пользователя и несёт те же риски, что и запуск любого локального кода. Ознакомьтесь с **[DISCLAIMER.md](../DISCLAIMER.md)** в корне репозитория.

## Что именно подменяется

Виджет из `project-widget/` занимает **всю область выбранного проекта** (контент под строкой вкладок `simple_game | simple_service | …`). Строка вкладок **по проектам** и левая глобальная панель SSH **не** подменяются.

- При **успешной** загрузке кастома **нет** встроенных у ядра вкладок «Метрики» и «Health» — только то, что вернул ваш `build` или класс **`CRMProjectTab`**.
- При **ошибке** загрузки или отсутствии точки входа CRM подставляет **fallback**: прежний интерфейс из **трёх вкладок** (Настройки с `ProjectSettingsCore`, Метрики и Health — заглушки).

## Зарезервированные имена

| Элемент | Значение |
|--------|----------|
| Каталог | `Projects/<id>/project-widget/` |
| Файл | **`base_project_widget.py`** (`crm.project_widget_loader.RESERVED_WIDGET_FILE`) |
| Функция (приоритет 1) | **`build`** (`ENTRYPOINT_NAME`) |
| Класс (приоритет 2) | **`CRMProjectTab`** (`RESERVED_CLASS_NAME`) |

Если в модуле есть **и** вызываемый `build`, **и** класс `CRMProjectTab`, используется **`build`**.

## Сигнатура `build`

```text
build(parent: QWidget, project_id: str, main_window) -> QWidget
```

- **`parent`** — родитель: контейнер области проекта (`_ProjectInner`).
- **`project_id`** — идентификатор проекта (имя каталога в `Projects/`).
- **`main_window`** — `MainWindow`: тосты, прогресс, `current_locale()`, подсветка SSH и т.д.

Возвращаемый **`QWidget`** — корень UI проекта (вкладки внутри, `QSplitter`, 3D, пустая панель — на усмотрение проекта).

## Сигнатура `CRMProjectTab`

Класс **`CRMProjectTab`** должен наследовать **`QWidget`** и поддерживать конструктор:

```text
CRMProjectTab(project_id: str, main_window, parent=None)
```

`main_window` — тот же экземпляр, что и в `build`.

## Поведение при ошибке

При отсутствии файла, ошибке импорта/исполнения, если `build` не вызываемый или вернул не `QWidget`, если нет пригодного класса `CRMProjectTab` или конструктор выбросил исключение — в лог пишется причина и показывается **`LegacyProjectTabsWidget`** (три вкладки, как в старом CRM).

## Интеграция SSH

`MainWindow` при валидации деплоя вызывает на области проекта **`mark_ssh_field_error` / `clear_ssh_field_error`**:

1. Если у корневого виджета есть эти методы — они вызываются.
2. Иначе выполняется **`findChild(ProjectSettingsCore)`** и делегирование в ядро.

Кастом может встроить `ProjectSettingsCore` или реализовать свои поля SSH и обработчики.

## Локализация (текущая версия)

Переключение **ru/en** в CRM **не обязано** затрагивать кастомный виджет: строки в `project-widget` могут быть захардкожены или использовать свою систему i18n. Если нужна реакция на смену языка CRM, реализуйте у корня **`apply_language()`** — тогда `MainWindow.apply_language()` вызовет его при смене локали. Без этого метода кастом при смене языка в CRM **не** обновляется автоматически.

## Обязательно (согласованность с CRM)

- Не обходить **`config_store`** при изменении **`project_profile.json`**.
- Операции сохранения профиля, **sync/push**, **SSH pull + restart** в смысле канона CRM — через **`ProjectSettingsCore`** (или тот же публичный путь), если вы не реализуете полностью свой пайплайн осознанно.
- Не блокировать GUI длительными операциями в **`build`** / **`__init__`** класса — вынести в поток/`QTimer` при необходимости.

## Запрещено / не рекомендуется

- Импорт из чужих монолитов репозитория вне пакета `crm/` без контроля окружения.
- Выполнение кода из сети при открытии проекта без явного действия пользователя.
- Скрытие кнопок деплоя/SSH без замены UX.

## Разрешено

- Любая визуальная структура внутри области проекта: без вкладок, один или несколько `QTabWidget`, `QSplitter`, сторонние виджеты (VTK и т.д. — зависимости только в окружении пользователя).
- Дополнительные модули в `project-widget/` и `import` соседних файлов (см. [project_widget_setup_and_git.md](project_widget_setup_and_git.md)).

## Ручная проверка

1. Запуск из корня: `.venv/bin/python3 main.py`.
2. Проект с валидным `project-widget` — одна «полоса» вашего UI без встроенных Метрик/Health.
3. Переименовать `base_project_widget.py` — снова три вкладки CRM (fallback).
